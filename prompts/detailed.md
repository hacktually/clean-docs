# Level of Detail

- Explain the *purpose* of the file before diving into specifics with one to two sentences.
- For every exported function/class: document parameters, return values, and thrown errors/exceptions
- Include at least one realistic usage example per major export. Not just a signature.
- Skip line-by-line explanations of trivial internal logic (e.g. simple loops, obvious variable assignments).
- Call out edge cases, gotchas, or non-obvious behavior explicitly (e.g. "Note: this mutates the input array").
- Prefer concrete examples over abstract descriptions.
