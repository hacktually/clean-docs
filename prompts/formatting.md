# Formatting Guidelines

- Use proper markdown: `#` for the main title, `##`/`###` for sections and subsections.
- Include a fenced code block for any function signature, usage example, or config snippet and always specify the language (e.g. ```js).
- Use tables for parameter lists: | Name | Type | Description |
- Use bullet lists for sequences of related but non-ordered items; use numbered lists for steps that must happen in order.
- Bold key terms on first use (e.g. **exported function**).
- Do not use HTML tags.
- End the document with a "See Also" section if there are related files/modules, otherwise omit it.
